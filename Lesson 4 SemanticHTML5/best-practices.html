HTML 5 Best Practices

  • Declare a doctype and language
  • Stick to lowercase for elements and file names
  • Avoid spaces in directories and filenames. Use a dash "-" instead.
  • Use the html, head, and body tags even though you don't have to
  • Do not use HTML for formatting, like with the <br> or &nbsp;. That is what padding and margins in CSS are form.
  • Close tags like &less <p> even though you don't have to
  • Use semantic elements where they make sense, but don't force it.
  • Use double quotes for values, like id="left-column", it's just sloppy
  • Always use the "alt" atribute for images for screen readers, or incase the image doesn't load. Plus it's good for SEO
  • Use comments where
  • Load CSS and JavaScript at the bottom of your file so the main content can load first
  • BE CONSISTENT in naming conentions, cassing, dashes or underscores, indentation, etc.
